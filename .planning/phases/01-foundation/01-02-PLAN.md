  <implementation>
    Follow the module-level function pattern from services/ibge.py and services/report.py — NO class.

    Mandatory implementation decisions (from research, non-negotiable):
    1. Import PLANILHA_EMPRESAS at module level: `from config import PLANILHA_EMPRESAS, PLANILHA_COL_COD, PLANILHA_COL_ANALISTA`
       (monkeypatch.setattr requires the name to live in the module's namespace)
    2. Always use `load_workbook(path, read_only=True, data_only=True)`
    3. Always call `wb.close()` in a `finally` block — never store wb between calls
    4. Catch PermissionError as a plain Python built-in (NOT inside openpyxl's exception clause — it raises before openpyxl runs)
    5. COD column validation: header_map["COD"] must equal PLANILHA_COL_COD (0)
       ANALISTA column validation: header_map["ANALISTA"] must equal PLANILHA_COL_ANALISTA (3)
    6. MUNICIPIO column position: discovered dynamically via header_map — NOT hardcoded index
       Also check for "MUNICÍPIO" (with accent) as fallback: `header_map.get("MUNICIPIO") or header_map.get("MUNICÍPIO")`
    7. GOIÂNIA filter: `str(municipio).strip().upper() == "GOIÂNIA"` (accent preserved — "GOIANIA" without accent is NOT a match per PLAN-02)
    8. COD type safety: always `str(cod).strip()` — Excel stores numeric cells as int
    9. Skip rows where cod is None or falsy

    Recommended internal structure (single private helper — re-opens workbook each call, avoids iter_rows exhaustion pitfall):
    ```python
    def _load_goiania_rows() -> list[dict]:
        # opens, validates headers, filters, closes, returns list[dict]
        # Each dict: {"cod": str, "analista": str}
    ```
    Both public functions call `_load_goiania_rows()` independently (opens workbook fresh each time).

    Error messages must be in Portuguese and name the cause clearly:
    - FileNotFoundError message: must include path and mention "G:" or "drive"
    - PermissionError message: must include "aberta" or "bloqueada" so test assertion passes
    - BadZipFile/InvalidFileException message: must include "corrompido" or "inválido"
    - Missing COD header: mention "COD" and "coluna A"
    - Missing ANALISTA header: mention "ANALISTA" and "coluna D"
    - Missing MUNICIPIO header: mention "MUNICIPIO"
